
================================================================================
PROJECT AUDIT REPORT: GHP ATTENDANCE & OT SYSTEM
Report Date: 2026-04-10
================================================================================

[หมวดที่ 1: ความปลอดภัย (Security & Privacy)]
--------------------------------------------------------------------------------
1.1 Data Privacy (FIXED ✅)
    - แก้ไขแล้ว: เพิ่มการเช็คสิทธิ์ (check_time_permission) 
      สิทธิ์ดูประวัติถูกจำกัดให้เฉพาะเจ้าของข้อมูล หรือ Admin/HR เท่านั้น

1.2 Sensitive Endpoint Exposure (FIXED ✅)
    - แก้ไขแล้ว: เพิ่ม current_user (บังคับ Auth) ใน Endpoint /ot-rules 
      คนภายนอกไม่สามารถดึงข้อมูลกฎระดับ Metadata ของบริษัทได้อีกต่อไป

1.3 Input Sanitation (LOW)
    - ไฟล์: ทุกที่ที่มีการรับ TextArea (Notes, Reason)
    - ปัญหา: ยังไม่มีการกรอง HTML/JS Script (XSS Risk) 
      หากพนักงานใส่ Script แปลกๆ มา Admin ที่เปิดดูอาจโดนขโมยเซสชั่นได้

1.4 Server-Side GPS Validation (MEDIUM)
    - ไฟล์: backend/routers/attendance.py
    - ปัญหา: แม้จะเก็บพิกัด GPS แต่หลังบ้านยังไม่ได้ "คำนวณระยะห่าง (Distance Check)" จริงๆ
      พนักงานที่รู้วิธี Debug Browser อาจจะหลอกพิกัดพิกัดปลอมส่งมาได้


[หมวดที่ 2: ตรรกะทางธุรกิจ (Business Logic)]
--------------------------------------------------------------------------------
2.1 Lack of OT Approval Workflow (HIGH)
    - ไฟล์: ระบบภาพรวม
    - ปัญหา: มีคนขอ OT ได้ แต่ไม่มี UI หรือ Endpoint ให้ Admin กด "อนุมัติ" หรือ "ปฏิเสธ" 
      ทำให้คำขอค้างเป็น Pending ในฐานข้อมูลตลอดกาล

2.2 Overlapping OT Requests (MEDIUM)
    - ไฟล์: backend/routers/attendance.py
    - ปัญหา: พนักงานสามารถส่งคำขอ OT ในช่วงเวลาที่ซ้อนทับกันได้ (เช่น ขอ 17:00-19:00 ไปแล้ว 
      แต่ยังกดขอ 18:00-20:00 ซ้ำได้อีกรอบ)

2.3 Attendance vs OT Sync (LOW)
    - ไฟล์: backend/models/attendance.py
    - ปัญหา: ตาราง AttendanceLog กับ OTRequest ยังไม่มีความเกี่ยวข้องกัน (Relationship) 
      ควรมีการตรวจสอบว่าวันที่ขอ OT พนักงานได้มาเช็คอินจริงหรือไม่


[หมวดที่ 3: ประสิทธิภาพและการแสดงผล (UX & Performance)]
--------------------------------------------------------------------------------
3.1 Lack of Pagination (LOW)
    - ไฟล์: backend/routers/attendance.py (Endpoint: /me)
    - ปัญหา: ดึงข้อมูลประวัติทั้งหมด (.all()) หากข้อมูลผ่านไปหลายเดือน/ปี 
      หน้าประวัติพนักงานจะโหลดช้ามากบน iPad

3.2 Connection Resilience (LOW)
    - ไฟล์: frontend/src/components/user/AttendancePanel.vue
    - ปัญหา: ยังไม่มีระบบจัดการเมื่อ Internet ของพนักงานไม่เสถียร (Retry Logic)
